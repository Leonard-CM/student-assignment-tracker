# Template Analysis and Selection

The following table compares different GitHub project templates and evaluates their suitability for Agile development.

| Template | Columns / Workflow | Automation Features | Agile Suitability |
|---------|------------------|---------------------|------------------|
| Basic Kanban | To Do, In Progress, Done | No automation | Suitable for simple workflows but lacks advanced tracking |
| Automated Kanban | To Do, In Progress, Done | Automatically moves issues based on status | Highly suitable for Agile workflows |
| Bug Triage | New, Needs Review, Fixed | Auto-labeling and prioritization | Best for bug tracking only |
| Team Planning | Backlog, Ready, In Progress, Done | Supports planning and prioritization | Good for team coordination but more complex |

---

## Selected Template: Automated Kanban

The Automated Kanban template was selected as it best supports Agile development and aligns with the requirements of the Student Assignment Tracker System.

### Justification

- **Automation Support**: Automatically moves issues between columns, reducing manual effort.
- **Agile Alignment**: Supports iterative development and continuous delivery.
- **Traceability**: Integrates with GitHub Issues created in Assignment 6.
- **Workflow Visibility**: Clearly shows task progression across stages.

Compared to other templates:
- Basic Kanban lacks automation
- Bug Triage is limited to bug tracking
- Team Planning is unnecessarily complex

Therefore, Automated Kanban is the most suitable choice.
