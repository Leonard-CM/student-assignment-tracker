# Reflection

One of the main challenges in this assignment was prioritizing user stories effectively. Since I am working individually, I had to balance my own expectations with the system requirements and stakeholder needs defined earlier. 
Another challenge was estimating effort using story points. Without a team, it was difficult to determine accurate estimates, which required careful judgment based on complexity.
Additionally, translating technical requirements into user-friendly stories required a shift in thinking from system-focused to user focused design.
Balancing must have features for the MVP was also challenging, as including too many features would increase complexity and delay delivery.
Overall, this assignment improved my understanding of Agile methodology, particularly backlog prioritization, sprint planning, and delivering incremental value.
