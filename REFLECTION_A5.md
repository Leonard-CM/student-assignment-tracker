# Reflection

This reflection outlines the challenges faced when translating system requirements into use case models and test cases, and highlights key insights gained during the process.

This assignment focused on translating system requirements into use case models and test cases. One of the main challenges was ensuring that all use cases aligned directly with the functional requirements defined in the previous assignment.
Another challenge was identifying appropriate actors and defining their interactions with the system. While the student is the primary actor, it was important to include supporting stakeholders such as lecturers and administrators to reflect real-world usage.
Developing detailed use case specifications required careful thinking to ensure that all possible scenarios, including alternative flows, were considered. This helped in identifying potential system weaknesses early in the design phase. 
Creating test cases also presented challenges, particularly in ensuring that each requirement was properly validated. Writing both functional and non-functional test cases improved understanding of system quality attributes such as performance and security.
Overall, this assignment highlighted the importance of clear requirement translation and structured system modeling in software engineering.
